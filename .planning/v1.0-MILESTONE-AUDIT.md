## Cross-Phase Integration

### Critical Issues

**1. Stripe Webhook Blocked by Middleware**
- **Route:** `/api/webhooks/stripe`
- **Issue:** Auth middleware redirects all unauthenticated requests. Stripe webhooks carry no session.
- **Impact:** BILL-01, BILL-02, BILL-03, BILL-04 non-functional in production
- **Fix:** Add `/api/webhooks` to middleware allow-list

**2. OTP Email Verification Blocked**
- **Route:** `/auth/verify`
- **Issue:** `/auth/confirm` redirects OTP flow to `/auth/verify` which is not in middleware allow-list
- **Impact:** AUTH-01 partial (PKCE flow works, OTP flow broken)
- **Fix:** Add `/auth/` paths to middleware allow-list

### Moderate Issues

**3. Continuity Auditor Dead Code**
- `runContinuityAudit()` exported from `continuity-auditor.ts` but never imported by `chapter/route.ts`
- Full UI for 409 conflict handling exists in `use-chapter-stream.ts` and `chapter-panel.tsx`
- The `force` parameter is accepted but has no effect
- Feature is structurally complete but functionally disconnected

### Low Priority

**4. Landing Page Unreachable**
- Root `/` has a `LandingPage` component but middleware redirects unauthenticated visitors to `/login`
