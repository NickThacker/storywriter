Complete rewrite of `characters-step.tsx`. Remove ALL existing code (role picker, ROLES array, toggleRole logic). Build from scratch:

**Layout structure:**
```
Header: "Who are the key players?" + subtitle about naming characters
Warning banner (if 8+ characters): "Having too many characters can dilute your story"
[+ Add Character] button  |  [Sparkles Suggest Cast] button  |  [Wand2 Suggest Names] button

Character cards grid (1 col mobile, 2 col sm+):
  Each card:
    - Name input (prominent, at top) with X remove button
    - Expand/collapse toggle for optional fields
    - When expanded: appearance, personality, backstory, arc textareas
    - [Wand2 Flesh Out] button on each card (generates details from name)

Empty state: "Add at least one character to continue" message with Add Character button
Footer info: "{N} character(s) defined" count
```

**Component state:**
- `expandedCards: Set<number>` -- which card indices are expanded
- `loadingAction: string | null` -- tracks which AI action is in progress ('suggest-names' | 'flesh-out-{index}' | 'suggest-cast' | null)
- `suggestedNames: string[] | null` -- names from "Suggest Names" shown in a popover/dropdown for user to pick

**AI integration (all use fetch to /api/generate/character-assist):**

1. **"Suggest Names" button** (top-level):
   - Sets `loadingAction = 'suggest-names'`
   - Calls API with `{ action: 'suggest-names', genre, setting, tone, themes, existingCharacters: characters.map(c => c.name), count: 5 }`
   - On success, shows a small panel/list below the button with suggested names, each with a "+" button to add as a new character card
   - On error, show a brief inline error message

2. **"Flesh Out" button** (per card):
   - Sets `loadingAction = 'flesh-out-{index}'`
   - Calls API with `{ action: 'flesh-out', genre, setting, tone, themes, character: { name: characters[index].name, appearance: characters[index].appearance, personality: characters[index].personality } }`
   - On success, calls `updateCharacter(index, { appearance, personality, backstory, arc })` with the AI response
   - Auto-expands the card to show the filled details
   - On error, show inline error

3. **"Suggest Cast" button** (top-level):
   - Sets `loadingAction = 'suggest-cast'`
   - Calls API with `{ action: 'suggest-cast', genre, setting, tone, themes, existingCharacters: characters.map(c => c.name) }`
   - On success, calls `setCharacters([...characters, ...response.characters])` to append new characters
   - On error, show inline error

**Character card details:**
- Name input: `<input>` with `onChange` calling `updateCharacter(index, { name: value })`
- Remove button: X icon calling `removeCharacter(index)`
- Expand toggle: ChevronDown/ChevronUp icon toggling the index in `expandedCards`
- Expanded section uses `transition-all duration-200` for smooth open/close with max-height trick or conditional render
- Optional fields use `<textarea>` with 2-3 rows, placeholder text explaining what to write
  - Appearance: "What do they look like? (hair, build, distinguishing features)"
  - Personality: "How do they act? Their voice, mannerisms, temperament"
  - Backstory: "Where did they come from? Key life events"
  - Arc: "How should they change over the course of the story?"
- Flesh Out button shows Loader2 spinner while loading, disabled when loadingAction is set

**Add Character button:** Calls `addCharacter({ name: '' })` and auto-focuses the new name input. Use a ref or useEffect to focus.

**Validation visual:** If a character has an empty name and user has interacted, show a red border on the name input.

**Style:** Follow existing card patterns in the app. Use `border border-border rounded-lg p-4` for cards. Primary accent for interactive elements. Muted background for expanded sections.

Do NOT include any role or archetype UI per user decision.
