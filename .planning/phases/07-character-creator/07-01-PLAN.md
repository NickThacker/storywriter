```typescript
hydrateFromPrefill: (prefill) =>
  set((state) => {
    const normalized = { ...state, ...prefill }
    // Normalize old { role, archetype, name? } format to new { name, ... }
    if (Array.isArray(normalized.characters)) {
      normalized.characters = normalized.characters.map((c: any) => {
        if ('role' in c && !('name' in c && c.name)) {
          // Old format: use role as placeholder name
          return { name: c.name || c.role || 'Unnamed' }
        }
        // Already new format or has name
        return { name: c.name, appearance: c.appearance, personality: c.personality, backstory: c.backstory, arc: c.arc }
      })
    }
    return normalized
  }),
```

Create `src/app/api/generate/character-assist/route.ts` following the established API route pattern (see premise-prefill/route.ts for reference).

The route handles three actions via a single POST endpoint:

**Request interface:**
```typescript
interface CharacterAssistRequest {
  action: 'suggest-names' | 'flesh-out' | 'suggest-cast'
  genre?: string
  setting?: string
  tone?: string
  themes?: string[]
  character?: { name: string; appearance?: string; personality?: string }
  count?: number
  existingCharacters?: string[]
}
```

**Implementation:**

1. Authenticate user via Supabase (same pattern as premise-prefill)
2. Get API key from user_settings
3. If no API key, return mock data for each action type
4. Get model via `getModelForRole(userId, 'planner')`
5. Build action-specific prompts:

**suggest-names:** System prompt asks for `count` (default 5) genre-appropriate character names given genre/setting/tone/themes. Must avoid names in `existingCharacters`. Response format: `{ names: string[] }`

**flesh-out:** System prompt takes a character's name (and any existing details) plus genre context, generates appearance, personality, backstory, and arc suggestions. Response format: `{ appearance: string, personality: string, backstory: string, arc: string }`

**suggest-cast:** System prompt generates 3-5 characters fitting the genre/setting/themes. Must avoid duplicating names in `existingCharacters`. Response format: `{ characters: { name: string, appearance: string, personality: string, backstory: string, arc: string }[] }`

6. Call OpenRouter with `response_format: { type: 'json_object' }`, `temperature: 0.7` (creative), `max_tokens: 1024`
7. Parse and validate response JSON
8. Log via `logPrompt` (fire-and-forget, same pattern as other routes)
9. Return parsed result

**Mock data for no-API-key fallback:**
- suggest-names: `{ names: ["Elena", "Marcus", "Zara", "Felix", "Ava"] }`
- flesh-out: `{ appearance: "...", personality: "...", backstory: "...", arc: "..." }` with placeholder text
- suggest-cast: `{ characters: [{ name: "Elena", ... }, { name: "Marcus", ... }, { name: "Zara", ... }] }`

Include a 30-second AbortController timeout. Handle errors consistently with other routes (401, 400, 502, 504, 500).
