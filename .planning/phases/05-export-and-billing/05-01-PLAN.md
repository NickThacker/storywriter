Create SQL migration `00005_billing_and_token_tracking.sql` with:

1. **`token_usage` table** -- one row per generation request:
   ```sql
   create table if not exists token_usage (
     id               uuid primary key default gen_random_uuid(),
     user_id          uuid references auth.users(id) on delete cascade not null,
     project_id       uuid references projects(id) on delete cascade not null,
     chapter_number   integer not null,
     prompt_tokens    integer not null default 0,
     completion_tokens integer not null default 0,
     total_tokens     integer not null default 0,
     created_at       timestamptz default now()
   );
   ```

2. **Extend `user_settings`** with billing columns:
   - `stripe_customer_id text` (nullable)
   - `stripe_subscription_id text` (nullable)
   - `token_budget_total integer not null default 0`
   - `token_budget_remaining integer not null default 0`
   - `credit_pack_tokens integer not null default 0`
   - `billing_period_end timestamptz` (nullable)
   Use `alter table ... add column if not exists` for each.

3. **Update `subscription_tier` constraint** to allow new tier names:
   ```sql
   alter table user_settings drop constraint if exists user_settings_subscription_tier_check;
   alter table user_settings add constraint user_settings_subscription_tier_check
     check (subscription_tier in ('none', 'hosted', 'starter', 'writer', 'pro'));
   ```

4. **`stripe_webhook_events` table** for idempotency:
   ```sql
   create table if not exists stripe_webhook_events (
     event_id     text primary key,
     processed_at timestamptz default now()
   );
   ```

5. **RLS policies**:
   - `token_usage`: enable RLS, add select policy for users reading their own rows (`(select auth.uid()) = user_id`). Inserts happen via service role (route handlers).
   - `stripe_webhook_events`: enable RLS, no user-facing policies (service role only).

6. **Indexes**:
   - `create index if not exists idx_token_usage_user on token_usage(user_id);`
   - `create index if not exists idx_token_usage_project on token_usage(project_id);`

NOTE: This migration must be applied manually in Supabase SQL Editor (not via CLI).

  completion_tokens: number
  total_tokens: number
  created_at: string
