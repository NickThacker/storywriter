  <name>Task 1: Create chapter assembly and export type definitions</name>
  <files>src/lib/export/assemble.ts</files>
  <action>
Create `src/lib/export/assemble.ts` with shared types and assembly logic:

```typescript
export interface ChapterContent {
  number: number
  title: string
  text: string
  isDraft: boolean   // true if approval_status !== 'approved'
}

export interface ExportOptions {
  projectTitle: string
  penName: string
  includeMode: 'approved' | 'all'  // which chapters to include
}

export interface AssembledBook {
  title: string
  author: string
  chapters: ChapterContent[]
}
```

The `assembleChapters` function:
1. Accept `projectId`, `includeMode` ('approved' | 'all'), `penName`, `projectTitle`
2. Create Supabase server client, authenticate user, verify project ownership (follow the `chapters.ts` pattern with `(supabase as any)`)
3. Fetch the project row for the title
4. Fetch the outline row for chapter titles (using `outlines` table, `chapters` JSON column)
5. Fetch chapter_checkpoints ordered by `chapter_number` ascending
6. Filter by `approval_status === 'approved'` if `includeMode === 'approved'`, otherwise include all
7. Map each checkpoint to `ChapterContent`, using the outline chapter title (fallback to `Chapter N` if no outline match)
8. Return `AssembledBook` object
9. Throw descriptive errors for auth failure, project not found, no chapters

Import `createClient` from `@/lib/supabase/server`. Use the same `(supabase as any)` cast pattern established in `chapters.ts`.

IMPORTANT: The assembly function should be callable from the route handler (server-side only). It does NOT need to be a server action.

**`src/lib/export/docx.ts`** — DOCX builder using `docx` library (v9.6.0):
- Import `Document, Packer, Paragraph, HeadingLevel, PageBreak, TextRun, AlignmentType` from `docx`
- `buildDocx(book: AssembledBook): Promise<Buffer>` function
- Structure:
  1. Title page: centered title (TITLE heading), centered author name, page break
  2. Manual Table of Contents: list of chapter titles as clickable bookmarks (NOT Word's auto-TOC field -- per research pitfall 4, auto-TOC requires Word "Update Field" which confuses users). Instead, create a simple list: `Chapter 1: Title`, `Chapter 2: Title`, etc. as HEADING_2, followed by a page break.
  3. For each chapter: HEADING_1 with chapter title (prefix "[DRAFT]" if `isDraft`), body paragraphs (split `text` on `\n\n` for paragraph breaks, `\n` for line breaks within paragraphs), page break between chapters
- Use `Packer.toBuffer(doc)` (NOT `toBlob` -- server-side only per research anti-pattern)
- Handle empty chapters gracefully (skip or show "[No content]")

**`src/lib/export/epub.ts`** — ePub builder using `epub-gen-memory`:
- `buildEpub(book: AssembledBook): Promise<Buffer>` function
- Use `epub-gen-memory` with `Options` type
- Set `title`, `author` (pen name), `tocTitle: 'Table of Contents'`, `version: 3`
- Map chapters to `content` array: `{ title: ch.title, content: htmlContent }`
- HTML-escape chapter text before inserting (per research pitfall 5): replace `<`, `>`, `&` with HTML entities
- Convert paragraphs to `<p>` tags (split on `\n\n`)
- If `isDraft`, prepend `<p><em>[DRAFT]</em></p>` to chapter content
- Return `await epub(options)` as Buffer

Import `AssembledBook` from `./assemble` in both files.

<task type="auto">
  <name>Task 3: Create hand-generated format builders (RTF and TXT)</name>
  <files>src/lib/export/rtf.ts, src/lib/export/txt.ts</files>
  <action>
Both builders are hand-generated (no library dependency beyond Node.js Buffer).

**`src/lib/export/rtf.ts`** — Hand-generated RTF (no library):
- `buildRtf(book: AssembledBook): Buffer` function (sync, no await needed)
- RTF structure (per research Pattern 7):
  1. RTF header: `{\rtf1\ansi\deff0{\fonttbl{\f0 Times New Roman;}}`
  2. Title page: centered, large font (`\fs48\qc`), author below, `\page`
  3. Each chapter: bold heading (`\b Chapter N: Title\b0`), body text (`\fs24\ql`), `\page` between chapters
  4. Close `}`
- `escapeRtf(text: string): string` helper: escape `{`, `}`, `\` with backslash prefix; replace non-ASCII characters with `\uNNNN?` (Unicode RTF escape)
- Draft marker: prepend `[DRAFT] ` to chapter title if isDraft
- Return `Buffer.from(rtf, 'utf-8')`

**`src/lib/export/txt.ts`** — Plain text (no library):
- `buildTxt(book: AssembledBook): Buffer` function (sync, no await needed)
- Structure:
  1. Title line centered-ish: `=== TITLE ===`, `by AUTHOR`, blank lines
  2. `--- TABLE OF CONTENTS ---`, list of `Chapter N: Title`
  3. Each chapter: `=== Chapter N: Title ===` (with [DRAFT] prefix if applicable), blank line, text, two blank lines between chapters
- Return `Buffer.from(text, 'utf-8')`

Import `AssembledBook` from `./assemble` in both files.

<task type="auto">
  <name>Task 4: Create unified export API route handler</name>
  <files>src/app/api/export/[projectId]/route.ts</files>
  <action>
Create a GET route handler at `src/app/api/export/[projectId]/route.ts`:

1. Extract `projectId` from params, `format` from query param (`?format=docx|epub|rtf|txt`, default `txt`)
2. Extract `include` from query param (`?include=approved|all`, default `approved`)
3. Extract `penName` from query param (`?penName=Author+Name`, default empty string)
4. Authenticate user via Supabase server client
5. Call `assembleChapters(projectId, includeMode, penName, projectTitle)` -- the assembly function handles auth and project lookup
6. Switch on format:
   - `docx` -> `buildDocx(book)`, content-type: `application/vnd.openxmlformats-officedocument.wordprocessingml.document`
   - `epub` -> `buildEpub(book)`, content-type: `application/epub+zip`
   - `rtf` -> `buildRtf(book)`, content-type: `application/rtf`
   - `txt` -> `buildTxt(book)`, content-type: `text/plain; charset=utf-8`
   - Invalid format -> 400 error
7. Generate filename: sanitize project title to slug (`title.toLowerCase().replace(/[^a-z0-9]+/g, '-')`) + `-novel.{ext}`
8. Return `new Response(buffer, { headers: { 'Content-Type': ..., 'Content-Disposition': 'attachment; filename="..."', 'Content-Length': String(buffer.byteLength) } })`
9. Wrap in try/catch for graceful error handling (return JSON error response on failure)

Set `export const dynamic = 'force-dynamic'` at top to prevent caching.

IMPORTANT: Do NOT use `export const runtime = 'edge'` -- the `docx` and `epub-gen-memory` libraries need Node.js runtime (Buffer, etc.).
