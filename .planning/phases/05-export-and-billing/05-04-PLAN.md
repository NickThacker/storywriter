**Create `src/actions/billing.ts`** as a server action module (`'use server'`):

1. **`createCheckoutSession(priceId: string)`**:
   - Import `stripe` from `@/lib/stripe/client`
   - If `stripe` is null, return `{ error: 'Stripe not configured' }`
   - Authenticate user via Supabase
   - Look up or create Stripe customer: check `user_settings.stripe_customer_id`, if null create via `stripe.customers.create({ email: user.email, metadata: { supabase_user_id: user.id } })` and save to `user_settings`
   - Create checkout session with `mode: 'subscription'`, `customer: stripeCustomerId`, `line_items: [{ price: priceId, quantity: 1 }]`, `success_url: ${origin}/settings?billing=success`, `cancel_url: ${origin}/settings?billing=cancelled`, `metadata: { userId: user.id }`
   - Return `{ url: session.url }` for redirect
   - Use `(await headers()).get('origin')` for origin

2. **`createCreditPackSession(priceId: string)`**:
   - Same auth + customer lookup as above
   - Create checkout session with `mode: 'payment'` (one-time), same customer, success/cancel URLs
   - Include `metadata: { userId: user.id, type: 'credit_pack' }`
   - Return `{ url: session.url }`

3. **`createPortalSession()`**:
   - Authenticate, get `stripe_customer_id` from user_settings
   - If no customer ID, return `{ error: 'No active subscription' }`
   - Create portal session via `stripe.billingPortal.sessions.create({ customer: customerId, return_url: ${origin}/settings })`
   - Return `{ url: session.url }`

4. **`getBillingStatus()`**:
   - Authenticate, fetch user_settings
   - If `openrouter_api_key` is set, return `BillingStatus` with `isByok: true` and zero budgets
   - Otherwise return full billing status with tier, budgets, credit pack tokens, period end, usage percent
   - Usage percent = `(budgetTotal - budgetRemaining) / budgetTotal * 100` (handle zero division)

**POST handler:**
1. Read raw body as text: `const body = await req.text()` (CRITICAL: NOT `req.json()` -- per research pitfall 1, JSON parsing destroys the raw body needed for signature verification)
2. Get Stripe signature: `req.headers.get('stripe-signature')`
3. Import `stripe` from `@/lib/stripe/client`. If null, return 500.
4. Verify signature: `stripe.webhooks.constructEvent(body, sig, process.env.STRIPE_WEBHOOK_SECRET!)`
5. Catch verification error -> return 400

**Idempotency check** (per research pitfall 2):
6. Create Supabase service role client (use `createClient` from `@supabase/supabase-js` directly with `SUPABASE_SERVICE_ROLE_KEY` -- NOT the user-scoped client, because webhooks have no user session)
7. Check `stripe_webhook_events` table for existing `event.id`
8. If already processed, return 200 (skip)
9. Insert `event.id` into `stripe_webhook_events`

**Event handling:**
```typescript
switch (event.type) {
  case 'checkout.session.completed':
    // Determine if subscription or credit pack from metadata
    const session = event.data.object
    if (session.mode === 'subscription') {
      await handleSubscriptionCheckout(session, supabase)
    } else if (session.metadata?.type === 'credit_pack') {
      await handleCreditPackPurchase(session, supabase)
    }
    break

  case 'customer.subscription.updated':
    await handleSubscriptionUpdate(event.data.object, supabase)
    break

  case 'customer.subscription.deleted':
    await handleSubscriptionDeleted(event.data.object, supabase)
    break

  case 'invoice.paid':
    await handleInvoicePaid(event.data.object, supabase)
    break
}
```

**Handler implementations:**

- `handleSubscriptionCheckout`: Look up user by `session.metadata.userId` or `session.customer`. Get the subscription's price ID. Find matching tier from `getTierByStripePriceId()`. Update `user_settings`: `subscription_tier`, `stripe_subscription_id`, `token_budget_total` (from tier config), `token_budget_remaining` (same as total), `billing_period_end`.

- `handleCreditPackPurchase`: Look up credit pack by line_items price. Add tokens to `credit_pack_tokens` column: `credit_pack_tokens + pack.tokens`.

- `handleSubscriptionUpdate`: Get updated subscription object. Find tier from price ID. Update `user_settings.subscription_tier`. If downgrade, adjust `token_budget_total` (but don't reduce `token_budget_remaining` below 0).

- `handleSubscriptionDeleted`: Set `subscription_tier = 'none'`, `token_budget_total = 0`, `token_budget_remaining = 0`, `stripe_subscription_id = null`.

- `handleInvoicePaid`: Reset `token_budget_remaining = token_budget_total` for the next billing period. Update `billing_period_end` from the invoice's `period_end`.
