**`src/lib/validations/auth.ts`** — Zod schemas:
- `loginSchema`: email (string, email format), password (string, min 8 chars)
- `signUpSchema`: email (string, email format), password (string, min 8 chars, max 72 chars), confirmPassword (string) — with `.refine()` to check passwords match
- `resetPasswordSchema`: email (string, email format)
- `updatePasswordSchema`: password (string, min 8 chars, max 72 chars), confirmPassword (string) — with `.refine()` for match

**`src/actions/auth.ts`** — Server Actions using `createClient` from `@/lib/supabase/server`:

1. `signIn(prevState: any, formData: FormData)` — Validates with `loginSchema`. Calls `supabase.auth.signInWithPassword()`. On success, redirects to `/dashboard`. On error, returns `{ error: string }`.

2. `signUp(prevState: any, formData: FormData)` — Validates with `signUpSchema`. Calls `supabase.auth.signUp()` with `emailRedirectTo: ${NEXT_PUBLIC_SITE_URL}/auth/confirm`. On success, creates a `user_settings` row (per research Open Question 4 — create in signup action, not DB trigger): insert into `user_settings` with `user_id` set to the new user's ID. Returns `{ success: 'Check your email to confirm your account' }`. On error, returns `{ error: string }`.

3. `signOut()` — Calls `supabase.auth.signOut()`. Redirects to `/login`.

4. `resetPassword(prevState: any, formData: FormData)` — Validates email with `resetPasswordSchema`. Calls `supabase.auth.resetPasswordForEmail()` with `redirectTo: ${NEXT_PUBLIC_SITE_URL}/auth/confirm?next=/auth/reset-password`. Returns success or error message.

5. `updatePassword(prevState: any, formData: FormData)` — Validates with `updatePasswordSchema`. Calls `supabase.auth.updateUser({ password })`. On success, redirects to `/dashboard`. On error, returns `{ error: string }`.

All actions must use the `'use server'` directive. All must call `supabase.auth.getUser()` where auth is required (updatePassword). Use `redirect()` from `next/navigation` for navigation (redirect throws, so call it AFTER any try/catch blocks — do not wrap redirect in try/catch).

**`src/app/(auth)/auth/confirm/route.ts`** — GET route handler for email confirmation and password reset:
- Extracts `token_hash`, `type`, and `next` from search params
- Creates a Supabase server client
- Calls `supabase.auth.verifyOtp({ type, token_hash })`
- On success, redirects to `next` param (defaults to `/dashboard`)
- On error, redirects to `/login?error=invalid_token`
- Follow the exact pattern from the research doc (Pattern: Auth Callback Route)

**`src/app/(auth)/auth/reset-password/page.tsx`** — Password update page:
- Only accessible after clicking the password reset email link (token already verified by confirm route)
- Renders a form with two password fields (new password, confirm password)
- Submits to `updatePassword` server action
- Shows success/error feedback
- Uses shadcn Input, Button, Card components
