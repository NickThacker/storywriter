**`src/lib/n8n/client.ts`:**

```typescript
// Server-side only — never import in Client Components
// N8N_BASE_URL and N8N_WEBHOOK_SECRET must NOT have NEXT_PUBLIC_ prefix
```

Environment variables used (server-only, no NEXT_PUBLIC_ prefix):
- `N8N_BASE_URL` — The internal/private URL of the n8n instance (e.g., `http://n8n.railway.internal:5678` or `http://localhost:5678` for local dev)
- `N8N_WEBHOOK_SECRET` — Random string shared between Next.js and n8n

Exports:

1. `triggerN8nWorkflow(webhookPath: string, payload: Record<string, unknown>): Promise<unknown>` — Makes a POST request to `${N8N_BASE_URL}/webhook/${webhookPath}` with:
   - `Content-Type: application/json` header
   - `X-Webhook-Secret: ${N8N_WEBHOOK_SECRET}` header
   - JSON body from `payload`
   - Throws an `N8nError` (custom error class) on non-2xx responses, including the status code and response body
   - Includes a 30-second timeout via `AbortSignal.timeout(30000)`

2. `isN8nConfigured(): boolean` — Returns true if both `N8N_BASE_URL` and `N8N_WEBHOOK_SECRET` are set. Used to conditionally show/hide n8n-dependent features and to fail gracefully in dev without n8n running.

3. `class N8nError extends Error` — Custom error with `status: number` and `body: string` properties.

Add a JSDoc comment at the top explaining:
- This module is server-side only
- The n8n instance validates the shared secret via an IF node (first node after webhook trigger)
- IF node checks: `{{ $request.headers['x-webhook-secret'] === $env.WEBHOOK_SECRET }}`
- If false, n8n responds with 401
