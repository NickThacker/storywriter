  <action>
Create the three Supabase client files following the exact `@supabase/ssr` patterns from the research doc:

**`src/lib/supabase/client.ts`** — Browser client using `createBrowserClient`. Exports a `createClient()` function. Used only in Client Components.

**`src/lib/supabase/server.ts`** — Server client using `createServerClient` with `cookies()` from `next/headers`. Exports an async `createClient()` function. Used in Server Components, Server Actions, and Route Handlers.

**`src/lib/supabase/middleware.ts`** — Exports an `updateSession(request)` function that creates a server client, calls `supabase.auth.getUser()` (NOT `getSession()` — per research anti-pattern warning), and handles cookie pass-through. This is the session refresh logic.

**`middleware.ts`** (root level) — Imports `updateSession` from `src/lib/supabase/middleware.ts`. Calls it on every request. Adds redirect logic: if no user and path is not `/login` and not `/auth/confirm`, redirect to `/login`. Uses the Supabase-recommended matcher that excludes `_next/static`, `_next/image`, `favicon.ico`, and image file extensions.

**`src/app/(auth)/layout.tsx`** — Minimal layout for auth pages. No navigation chrome. Centered content. If user is already authenticated (check via `getUser()`), redirect to `/dashboard`.

**`src/app/(dashboard)/layout.tsx`** — Authenticated app shell layout. Checks `getUser()` — if no user, redirect to `/login`. Renders a basic app shell with a top navigation bar containing: app name "StoryWriter" (left), navigation links for "Dashboard" and "Settings" (center-left), and a "Sign out" button (right). The sign out button calls the `signOut` server action (which will be created in Plan 02 — for now, create a placeholder that does nothing). Children render below the nav bar.

**`src/types/database.ts`** — TypeScript types matching the database schema. Define interfaces for:
- `UserSettings` (id, user_id, openrouter_vault_id, subscription_tier, created_at, updated_at)
- `UserModelPreference` (id, user_id, task_type, model_id, updated_at)
- `Project` (id, user_id, title, status, genre, word_count, chapter_count, chapters_done, story_bible, created_at, updated_at)
- Export a `Database` type following Supabase's generated types pattern (Tables with Row, Insert, Update variants)
- `ProjectStatus` type: `'draft' | 'writing' | 'complete'`
- `SubscriptionTier` type: `'none' | 'hosted'`
- `TaskType` type: `'outline' | 'prose' | 'editing'`

  <name>Task 3: Create database migration with full schema, RLS policies, Vault functions, and triggers</name>
  <files>

  <action>
Create the SQL migration file with the complete Phase 1 database schema. This single migration establishes ALL tables needed through Phase 2+ (story bible columns seeded now per STATE.md architectural decision).

**Tables:**

1. `user_settings` — 1:1 with `auth.users`:
   - `id uuid primary key default gen_random_uuid()`
   - `user_id uuid references auth.users(id) on delete cascade not null unique`
   - `openrouter_vault_id uuid` (nullable — null means no BYOK key)
   - `subscription_tier text not null default 'none'` (check constraint: 'none' or 'hosted')
   - `created_at timestamptz default now()`
   - `updated_at timestamptz default now()`

2. `user_model_preferences` — Per-task model overrides:
   - `id uuid primary key default gen_random_uuid()`
   - `user_id uuid references auth.users(id) on delete cascade not null`
   - `task_type text not null` (check constraint: 'outline', 'prose', or 'editing')
   - `model_id text not null` (OpenRouter model identifier string)
   - `updated_at timestamptz default now()`
   - `unique(user_id, task_type)`

3. `projects` — Novel projects:
   - `id uuid primary key default gen_random_uuid()`
   - `user_id uuid references auth.users(id) on delete cascade not null`
   - `title text not null default 'Untitled Novel'`
   - `status text not null default 'draft'` (check constraint: 'draft', 'writing', 'complete')
   - `genre text`
   - `word_count integer not null default 0`
   - `chapter_count integer not null default 0`
   - `chapters_done integer not null default 0`
   - `story_bible jsonb not null default '{}'::jsonb` (seeded for Phase 2)
   - `created_at timestamptz default now()`
   - `updated_at timestamptz default now()`

**RLS Policies (ALL tables):**
- Enable RLS on every table immediately after creation
- For each table, create 4 policies: select, insert, update, delete — all scoped to `(select auth.uid()) = user_id`
- Use the `(select auth.uid())` pattern (wrapped in subquery) for performance per Supabase docs

**Triggers:**
- Create a reusable `update_updated_at()` function that sets `new.updated_at = now()`
- Attach `before update` trigger to `user_settings`, `user_model_preferences`, and `projects`

**Vault Functions (SECURITY DEFINER):**

1. `upsert_user_api_key(p_user_id uuid, p_api_key text) returns uuid` — Creates or updates an OpenRouter API key in Vault. If user already has a `openrouter_vault_id`, updates the existing secret. Otherwise creates a new one via `vault.create_secret()` and stores the UUID in `user_settings`. Uses `security definer` and sets `search_path = public, vault`.

2. `get_decrypted_api_key(p_vault_id uuid) returns text` — Retrieves a decrypted API key from `vault.decrypted_secrets` by ID. Uses `security definer`. Returns null if not found.

**Indexes:**
- `projects_user_id_updated_at_idx` on `projects(user_id, updated_at desc)` — for dashboard sorting
- `user_model_preferences_user_id_idx` on `user_model_preferences(user_id)` — for settings page load
