**`src/lib/validations/project.ts`** — Zod schemas:
- `createProjectSchema`: title (string, min 1, max 200, trimmed), genre (string, optional)
- `updateProjectSchema`: partial object with optional title, genre, status, word_count, chapter_count, chapters_done, story_bible — all validated to correct types

**`src/actions/projects.ts`** — Server Actions:

1. `createProject(prevState: any, formData: FormData)`:
   - Validates with `createProjectSchema`
   - Gets user via `supabase.auth.getUser()` — return `{ error: 'Unauthorized' }` if no user
   - Inserts into `projects` table with `user_id: user.id` and parsed fields
   - Calls `revalidatePath('/dashboard')` to refresh the project list
   - Returns `{ projectId: data.id }` on success or `{ error: string }` on failure

2. `deleteProject(projectId: string)`:
   - Gets user via `getUser()` — unauthorized check
   - Deletes from `projects` where `id = projectId` (RLS enforces ownership automatically)
   - Calls `revalidatePath('/dashboard')`
   - Returns `{ success: true }` or `{ error: string }`

3. `updateProject(projectId: string, updates: Partial<Project>)`:
   - Gets user via `getUser()` — unauthorized check
   - Validates `updates` with `updateProjectSchema`
   - Updates `projects` row where `id = projectId` (RLS enforces ownership)
   - Calls `revalidatePath('/dashboard')`
   - Returns `{ success: true }` or `{ error: string }`
   - This is the auto-save target — called by debounced client code (PROJ-05)

All actions use `'use server'` directive. All authenticate via `getUser()`. No `getSession()`.
