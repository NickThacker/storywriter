**`src/lib/validations/settings.ts`** — Zod schemas:
- `apiKeySchema`: apiKey (string, min 10, starts with "sk-" — OpenRouter keys start with "sk-or-")
- `modelPreferenceSchema`: taskType (enum: outline, prose, editing), modelId (string, min 1)

**`src/actions/settings.ts`** — Server Actions:

1. `saveApiKey(prevState: any, formData: FormData)`:
   - Gets user via `getUser()`
   - Validates API key with `apiKeySchema`
   - Calls `supabase.rpc('upsert_user_api_key', { p_user_id: user.id, p_api_key: apiKey })`
   - Returns `{ success: true }` or `{ error: string }`
   - CRITICAL: The raw API key must never be returned to the client after saving

2. `testApiKey(apiKey: string)`:
   - Server-side function — validates API key by calling `https://openrouter.ai/api/v1/models` with `Authorization: Bearer ${apiKey}`
   - If response is 200, return `{ valid: true }`
   - If response is 401/403, return `{ valid: false, error: 'Invalid API key' }`
   - If fetch fails, return `{ valid: false, error: 'Connection failed' }`
   - Has a 10-second timeout

3. `deleteApiKey()`:
   - Gets user via `getUser()`
   - Looks up `openrouter_vault_id` from `user_settings`
   - If exists, deletes from `vault.secrets` via an RPC function or direct delete, then sets `openrouter_vault_id` to null in `user_settings`
   - Returns `{ success: true }` or `{ error: string }`

4. `getApiKeyStatus()`:
   - Gets user via `getUser()`
   - Queries `user_settings` for `openrouter_vault_id` and `subscription_tier`
   - If `openrouter_vault_id` is not null, retrieves the decrypted key via `get_decrypted_api_key` RPC and extracts only the last 4 characters
   - Returns `{ hasKey: boolean, keyHint: string | null, subscriptionTier: string }` — keyHint is the last 4 chars (e.g., "abcd"), NEVER the full key

5. `saveModelPreferences(preferences: { taskType: string; modelId: string }[])`:
   - Gets user via `getUser()`
   - Validates each preference with `modelPreferenceSchema`
   - Upserts into `user_model_preferences` for each task type (upsert on `user_id, task_type` unique constraint)
   - Returns `{ success: true }` or `{ error: string }`

6. `getModelPreferences()`:
   - Gets user via `getUser()`
   - Queries `user_model_preferences` for the user
   - Returns array of `{ taskType, modelId }` — fills in defaults from `RECOMMENDED_MODELS` for any task type not explicitly set

**`src/components/settings/api-key-form.tsx`** — Client Component:

Per user decision: "Dedicated settings page for BYOK OpenRouter key entry. 'Test connection' button validates key before saving. Key stored server-side only, displayed as masked with last 4 characters (e.g., ••••••••abcd)."

Props: `initialKeyStatus: { hasKey: boolean; keyHint: string | null; subscriptionTier: string }`

States:
- `hasKey` / `keyHint` from props (updated after save/delete)
- `isEditing` — toggled when user wants to enter/change key
- `testResult` — null / 'valid' / 'invalid' / 'testing'

UI when key EXISTS (`hasKey: true`):
- Shows masked display: "••••••••{keyHint}" (e.g., "••••••••abcd")
- "Change key" button (enters editing mode)
- "Remove key" button (red/destructive, calls `deleteApiKey`)
- Green checkmark or "Connected" indicator

UI when NO key or EDITING:
- Text input for API key (type="password" for security, but with a show/hide toggle)
- "Test Connection" button — calls `testApiKey` with the entered key
  - While testing: shows spinner, button disabled
  - On valid: green checkmark, "Key is valid" message
  - On invalid: red X, error message
- "Save Key" button — calls `saveApiKey` server action
  - Only enabled after successful test (or allow save without test with a warning)
  - After save: exits editing mode, updates masked display
- "Cancel" button — exits editing mode without saving
- Link: "Get an OpenRouter API key" → opens https://openrouter.ai/keys in new tab

Styling: shadcn Card for the section container, Input for the key field, Button for actions
