**`src/lib/validations/intake.ts`:**
Define a Zod schema for validating intake data before saving:

```typescript
import { z } from 'zod'

export const intakeDataSchema = z.object({
  path: z.enum(['wizard', 'premise']),
  genre: z.string().nullable(),
  themes: z.array(z.string()),
  characters: z.array(z.object({
    role: z.string(),
    archetype: z.string(),
    name: z.string().optional(),
  })),
  setting: z.string().nullable(),
  tone: z.string().nullable(),
  beatSheet: z.string().nullable(),
  targetLength: z.enum(['short', 'standard', 'epic']),
  chapterCount: z.number().int().min(5).max(60),
  premise: z.string().nullable(),
})

export type IntakeData = z.infer<typeof intakeDataSchema>
```

**`src/actions/intake.ts`:**
Mark as `'use server'`. Create server actions:

1. **`saveIntakeData(projectId: string, data: IntakeData)`** — Validates input with Zod. Authenticates user. Updates the project's `intake_data` jsonb column and also updates `genre` if set. Uses `(supabase as any)` pattern consistent with Phase 1. Returns `{ success: true }` or `{ error: string }`.

2. **`getIntakeData(projectId: string)`** — Fetches the project's `intake_data` column. Returns `{ data: IntakeData | null }` or `{ error: string }`. Used by the intake layout to hydrate the store with saved state.

Both actions verify the user owns the project before proceeding (check user_id matches auth.uid()).
