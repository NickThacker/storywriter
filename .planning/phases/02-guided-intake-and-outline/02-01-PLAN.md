  <name>Task 1: Create story bible database migration</name>
  <files>
    supabase/migrations/00002_story_bible_tables.sql
  </files>
  <action>
Create the Phase 2 database migration with four normalized tables. This replaces the flat `projects.story_bible` JSONB for primary story data (the JSONB column is kept as a metadata overflow bucket).

**`characters` table:**
- `id uuid primary key default gen_random_uuid()`
- `project_id uuid references projects(id) on delete cascade not null`
- `name text not null`
- `role text not null` â€” 'protagonist' | 'antagonist' | 'supporting' | 'minor'
- `one_line text` â€” card-level summary
- `appearance text`
- `backstory text`
- `personality text`
- `voice text`
- `motivations text`
- `arc text`
- `source text not null default 'ai'` â€” 'ai' | 'manual' (tracks origin per research Open Question 3; allows merge on outline approval without overwriting manual edits)
- `created_at timestamptz default now()`
- `updated_at timestamptz default now()`

**`locations` table:**
- `id uuid primary key default gen_random_uuid()`
- `project_id uuid references projects(id) on delete cascade not null`
- `name text not null`
- `description text`
- `significance text`
- `created_at timestamptz default now()`
- `updated_at timestamptz default now()`

**`world_facts` table:**
- `id uuid primary key default gen_random_uuid()`
- `project_id uuid references projects(id) on delete cascade not null`
- `category text not null` â€” 'timeline' | 'rule' | 'lore' | 'relationship'
- `fact text not null`
- `created_at timestamptz default now()`
- `updated_at timestamptz default now()`

**`outlines` table:**
- `id uuid primary key default gen_random_uuid()`
- `project_id uuid references projects(id) on delete cascade not null unique`
- `beat_sheet_id text not null` â€” 'save-the-cat' | 'three-act' | 'heros-journey' | 'romancing-the-beat'
- `target_length text not null` â€” 'short' | 'standard' | 'epic'
- `chapter_count integer not null`
- `chapters jsonb not null default '[]'::jsonb` â€” Array of chapter objects with number, title, summary, beats, characters_featured, beat_sheet_mapping, act
- `previous_chapters jsonb` â€” Snapshot of previous chapters on regeneration (per research Open Question 2: store data now, skip restore UI)
- `status text not null default 'draft'` â€” 'draft' | 'approved'
- `approved_at timestamptz`
- `created_at timestamptz default now()`
- `updated_at timestamptz default now()`
- `constraint outlines_status_check check (status in ('draft', 'approved'))`

**RLS Policies:** Enable RLS on all 4 tables. Use a single "for all" policy per table scoped via subquery to project owner:
```sql
create policy "Users can manage own project characters"
  on characters for all
  to authenticated
  using (
    (select auth.uid()) = (select user_id from projects where id = project_id)
  );
```
Repeat for locations, world_facts, outlines.

**Triggers:** Attach `update_updated_at()` (already defined in migration 00001) to all 4 tables as `before update` triggers.

**Indexes:**
- `characters_project_id_idx on characters (project_id)`
- `locations_project_id_idx on locations (project_id)`
- `world_facts_project_id_idx on world_facts (project_id)`

Also add `intake_data jsonb` column to `projects` table via ALTER TABLE â€” stores the wizard answers for outline regeneration and reference. This keeps intake data associated with the project.

```sql
alter table projects add column if not exists intake_data jsonb;
```
