<objective>
Build the AI outline generation pipeline: the SSE streaming route handler that calls OpenRouter directly, the client-side hook that consumes the stream, and the server actions for outline persistence. This is the core AI integration for Phase 2.

Purpose: OUTL-01 requires AI-generated outlines from intake data. The hybrid streaming pattern (Next.js Route Handler calling OpenRouter directly, NOT through n8n) avoids the n8n 30-second timeout constraint for a generation that may take 60-120s.

**n8n deviation note:** The user decision specifies "Multi-step n8n workflow" for AI generation. However, per RESEARCH.md findings, n8n's default webhook timeout (30s) is insufficient for outline generation (60-120s). Direct OpenRouter streaming from the Next.js Route Handler is the correct architecture. n8n remains used for lightweight tasks like premise-prefill (Plan 02-04) where latency is under 30s. This deviation is architecturally necessary and does not reduce observability — SSE events provide equivalent step visibility to the client.

Output: Complete outline generation pipeline from trigger to storage.
</objective>

**`src/app/api/generate/outline/route.ts`:**

CRITICAL: `export const dynamic = 'force-dynamic'` — prevents Vercel from caching the streaming response.

POST handler implementation:
1. Authenticate user via `createClient()` + `getUser()`. Return 401 if not authenticated.
2. Parse request body: `{ projectId: string, intakeData: IntakeData, direction?: string }`.
3. Retrieve API key server-side: query `user_settings` for `openrouter_api_key` (using `(supabase as any)` pattern). If Vault-backed, call the `get_decrypted_api_key` function. Return 400 if no key.
4. Retrieve user's preferred outline model: query `user_model_preferences` for `task_type = 'outline'`. Default to `'anthropic/claude-sonnet-4-5'` if none set.
5. Build prompt using `buildOutlinePrompt(intakeData)` or `buildRegeneratePrompt(intakeData, direction)` if direction is provided.
6. Call OpenRouter directly:
   ```typescript
   const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
     method: 'POST',
     headers: {
       'Authorization': `Bearer ${apiKey}`,
       'Content-Type': 'application/json',
       'HTTP-Referer': process.env.NEXT_PUBLIC_SITE_URL || 'http://localhost:3000',
       'X-Title': 'StoryWriter',
     },
     body: JSON.stringify({
       model: modelId,
       stream: true,
       messages: [
         { role: 'system', content: systemPrompt },
         { role: 'user', content: userPrompt },
       ],
       response_format: {
         type: 'json_schema',
         json_schema: { name: 'novel_outline', strict: true, schema: OUTLINE_SCHEMA },
       },
     }),
   })
   ```
7. If OpenRouter response not OK, return 502 with error message.
8. Stream the response through to the client as-is using a ReadableStream that pipes the OpenRouter response body.
9. Return with SSE headers: `Content-Type: text/event-stream; charset=utf-8`, `Cache-Control: no-cache, no-transform`, `Connection: keep-alive`, `Content-Encoding: none`.

**`src/hooks/use-outline-stream.ts`:**

Mark as `'use client'` compatible (it's a hook file, not a component, but uses React hooks).

Export `useOutlineStream(projectId: string)` hook:

Returns: `{ streamedContent: string, parsedOutline: GeneratedOutline | null, isStreaming: boolean, error: string | null, startStream: (intakeData, direction?) => Promise<void> }`

Implementation:
1. `startStream` POSTs to `/api/generate/outline` with `{ projectId, intakeData, direction }`.
2. Uses `response.body.pipeThrough(new TextDecoderStream()).getReader()` pattern.
3. Reads chunks in a while loop. For each chunk:
   - Split on newlines, process lines starting with `data: `
   - Skip `data: [DONE]` (completion signal) and lines starting with `: ` (OpenRouter heartbeat)
   - Parse JSON, extract `choices[0].delta.content`
   - Accumulate into a running string, update `streamedContent` state
   - Wrap JSON.parse in try/catch — skip partial chunks (per research Pitfall 3)
4. On stream completion, attempt to `JSON.parse` the full accumulated string into `GeneratedOutline`. Set `parsedOutline` state.
5. Handle errors: set `error` state, set `isStreaming` to false.

Mark as `'use server'`. Create server actions for outline persistence.

**`saveOutline(projectId: string, outlineData: GeneratedOutline, intakeData: IntakeData)`:**
1. Authenticate user, verify project ownership.
2. Check if an outline already exists for this project (unique constraint on project_id).
3. If exists: snapshot current `chapters` into `previous_chapters` (per research Open Question 2), then update with new data.
4. If not exists: insert new row into `outlines` table with `status: 'draft'`.
5. Set `beat_sheet_id` from `intakeData.beatSheet`, `target_length` from `intakeData.targetLength`, `chapter_count` from actual chapters length.
6. Store `outlineData.chapters` in the `chapters` jsonb column.
7. Also update the project title from `outlineData.title` and set `chapter_count` on the projects table.
8. Return `{ success: true, outlineId: string }` or `{ error: string }`.

**`getOutline(projectId: string)`:**
1. Authenticate user, verify project access.
2. Query `outlines` table for `project_id = projectId`.
3. Return `{ data: OutlineRow | null }` or `{ error: string }`.

**`updateOutlineChapter(projectId: string, chapterIndex: number, updates: Partial<OutlineChapter>)`:**
1. Authenticate user, verify project ownership.
2. Fetch current outline.
3. Update the specific chapter at `chapterIndex` in the `chapters` array.
4. Save back to database.
5. Return `{ success: true }` or `{ error: string }`.

Use `revalidatePath` for `/projects/[id]/outline` after mutations.
