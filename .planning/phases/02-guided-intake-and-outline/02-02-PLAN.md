Create the Zustand store for intake wizard state using the **vanilla store + React Context provider** pattern (mandatory for Next.js App Router per research — avoids global singleton SSR leak).

**`src/lib/stores/intake-store.ts`:**

Use `createStore` from `zustand/vanilla` (NOT the `create` shorthand — vanilla is required for the provider pattern).

```typescript
import { createStore } from 'zustand/vanilla'

export type BeatSheetId = 'save-the-cat' | 'three-act' | 'heros-journey' | 'romancing-the-beat'
export type NovelLength = 'short' | 'standard' | 'epic'

export interface IntakeState {
  // Path selection
  path: 'wizard' | 'premise' | null
  // Step data
  genre: string | null
  themes: string[]
  characters: { role: string; archetype: string; name?: string }[]
  setting: string | null
  tone: string | null
  beatSheet: BeatSheetId | null
  targetLength: NovelLength
  chapterCount: number
  premise: string | null
  // Navigation
  currentStep: number
  // Actions
  setPath: (path: 'wizard' | 'premise') => void
  setGenre: (genre: string) => void
  setThemes: (themes: string[]) => void
  addCharacter: (character: { role: string; archetype: string; name?: string }) => void
  removeCharacter: (index: number) => void
  setSetting: (setting: string) => void
  setTone: (tone: string) => void
  setBeatSheet: (id: BeatSheetId) => void
  setTargetLength: (length: NovelLength, chapters: number) => void
  setChapterCount: (count: number) => void
  setPremise: (text: string) => void
  hydrateFromPrefill: (prefill: Partial<IntakeState>) => void
  goToStep: (step: number) => void
  nextStep: () => void
  prevStep: () => void
  reset: () => void
}
```

Define `TOTAL_STEPS = 7` (path select, genre, themes, characters, setting, tone+beatsheet+length, review).

Implement all actions. `nextStep` increments `currentStep` (clamped to TOTAL_STEPS - 1). `prevStep` decrements (clamped to 0). `reset` restores all data fields to defaults.

Export `createIntakeStore`, `IntakeState`, and the type `IntakeStore = ReturnType<typeof createIntakeStore>`.

**`src/components/intake/intake-store-provider.tsx`:**

Mark as `'use client'`. Use the official Zustand + Next.js App Router pattern:

1. Create a React Context that holds the store instance (NOT the state).
2. `IntakeStoreProvider` component creates the store via `useRef` (ensures single instance per mount).
3. Accept optional `initialState?: Partial<IntakeState>` prop for server-side hydration of draft intake data.
4. Export `useIntakeStore<T>(selector: (state: IntakeState) => T): T` hook that reads from context and uses `useStore` with selector for efficient re-renders.
5. Throw a descriptive error if `useIntakeStore` is called outside the provider.

Do NOT use Zustand persist middleware — no localStorage. Wizard state is ephemeral per session, backed by DB on submit (per research anti-pattern: hydration mismatch from persist + SSR).
