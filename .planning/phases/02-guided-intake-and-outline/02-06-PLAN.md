**`src/components/outline/inline-editable.tsx`:**
Mark as `'use client'`. Reusable click-to-edit component (per research Open Question 1: use reveal-on-click input, NOT contentEditable).

Props: `value: string`, `onSave: (value: string) => void`, `as?: 'h2' | 'p' | 'span'` (determines rendered element), `className?: string`, `multiline?: boolean`

Behavior:
1. **View mode** (default): Renders text in the specified element. On click or Enter key, switches to edit mode. Shows a subtle edit icon on hover.
2. **Edit mode**: Renders an `<input>` (single line) or `<textarea>` (multiline) with the same styling. Pre-fills with current value. Focus auto-set.
3. **Save**: On blur or Enter (for input) or Ctrl+Enter (for textarea), calls `onSave(newValue)` and returns to view mode.
4. **Cancel**: On Escape, revert to original value and return to view mode.

This avoids contentEditable React issues (cursor position, paste handling, re-render conflicts) per research recommendation.
