**src/app/(dashboard)/onboarding/page.tsx** — Client Component, renders the correct step based on store.currentStep. Also renders the shared progress bar and step heading:
```typescript
'use client'
import { useVoiceWizardStore } from '@/components/onboarding/onboarding-store-provider'
import { TOTAL_VOICE_STEPS } from '@/lib/stores/voice-wizard-store'
import ProgressBar from '@/components/intake/progress-bar'
import { WritingSamplesStep } from '@/components/onboarding/writing-samples-step'
import { StylePreferencesStep } from '@/components/onboarding/style-preferences-step'
import { AnalysisResultsStep } from '@/components/onboarding/analysis-results-step'

const STEP_TITLES = ['Writing Samples', 'Style Preferences', 'Voice Analysis']

export default function OnboardingPage() {
  const currentStep = useVoiceWizardStore((s) => s.currentStep)

  return (
    <div className="mx-auto max-w-2xl space-y-6 py-8">
      <div>
        <h1 className="text-2xl font-bold tracking-tight">Author Voice Setup</h1>
        <p className="mt-1 text-muted-foreground">
          Provide writing samples so StoryWriter can match your style.
        </p>
      </div>
      <ProgressBar currentStep={currentStep} totalSteps={TOTAL_VOICE_STEPS} />
      <div className="text-sm font-medium text-muted-foreground">
        Step {currentStep} of {TOTAL_VOICE_STEPS} — {STEP_TITLES[currentStep - 1]}
      </div>
      {currentStep === 1 && <WritingSamplesStep />}
      {currentStep === 2 && <StylePreferencesStep />}
      {currentStep === 3 && <AnalysisResultsStep />}
    </div>
  )
}
```

**src/components/onboarding/style-preferences-step.tsx** — Client Component, Step 2:
Use CardPicker from `@/components/intake/card-picker` for each preference dimension. Six CardPicker groups:

1. **Tone** (`tonePreference`): options — "Literary & Lyrical", "Direct & Gritty", "Warm & Intimate", "Dark & Brooding", "Witty & Sharp"
2. **Pacing** (`pacingPreference`): options — "Fast & Propulsive", "Deliberate & Measured", "Variable (action + reflection)", "Slow-burn atmospheric"
3. **Dialogue Ratio** (`dialogueRatio`): options — "Heavy dialogue (character-driven)", "Balanced mix", "Sparse dialogue (prose-driven)", "Interior monologue focused"
4. **Thematic Tone** (`darkLightTheme`): options — "Dark & Complex", "Hopeful with tension", "Light & Uplifting", "Morally ambiguous"
5. **Point of View** (`povPreference`): options — "First person (intimate)", "Third person limited", "Third person omniscient", "Second person (experimental)"
6. **Diction Level** (`dictionLevel`): options — "Literary (elevated vocabulary)", "Contemporary & Accessible", "Stark & Minimalist", "Genre-specific register"

Each CardPicker calls `setStylePreference(key, value)` on selection. All are optional.

WizardNav at bottom: `canProceed` always true (preferences are optional). `onNext` saves wizard_step=3 to DB via `savePersona({ wizard_step: 3 })` then calls `nextStep()`.

**src/components/onboarding/analysis-results-step.tsx** — Client Component, Step 3:
This is the streaming analysis view. Mirrors the outline streaming view pattern.

On mount (useEffect with empty deps):
1. Collect all samples from store (pastedSamples + uploadedFileTexts) and preferences
2. POST to `/api/voice-analysis` with `{ samples, preferences }` — response is SSE stream
3. Read SSE stream chunks, accumulate raw text, attempt `parsePartialJSON` on each chunk
4. When analysis completes (stream ends), call `setAnalysisResult(result)` on the store and `savePersona({ ...result, analysis_complete: true, wizard_step: 3 })`

Display while streaming: show a card with animated "Analyzing your writing style..." with progressive reveal of the voice_description field as it appears in the partial JSON.

Display after complete:
- Voice Description section (large text)
- Style Descriptors section (grid of label/value pairs)
- Thematic Preferences section (grid of label/value pairs)
- "Download PDF Report" button — links to `/api/voice-report` (GET request, triggers file download using the anchor-based pattern from export dialog)
- "Go to Dashboard" button — navigates to `/dashboard`

Parse partial JSON using the same `parsePartialJSON` utility already in the codebase. Import from `@/components/outline/streaming-view` or wherever it exists — check the actual import path.

Handle error state: if the analysis fetch fails, show an error card with a "Try Again" button that re-triggers the effect.
