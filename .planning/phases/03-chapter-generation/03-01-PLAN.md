<task type="auto">
  <name>Task 1: Create useChapterStream hook</name>
  <files>src/hooks/use-chapter-stream.ts</files>
  <action>
Create a client-side hook that mirrors `useOutlineStream` but accumulates raw prose tokens (not JSON).

The hook must export `useChapterStream()` returning:
- `streamedText: string` — accumulated prose
- `isStreaming: boolean` — whether a stream is active
- `isPaused: boolean` — whether the user paused (stream aborted, text preserved)
- `error: string | null` — error message
- `wordCount: number` — live word count derived from streamedText
- `startStream(projectId: string, chapterNumber: number, adjustments?: string): Promise<void>`
- `pause(): void` — aborts the stream, sets isPaused true, preserves streamedText
- `resume(): void` — clears isPaused flag (caller re-triggers startStream for actual resumption)
- `stop(): void` — aborts the stream, clears isPaused

Implementation details:
1. Use `useState` for streamedText, isStreaming, isPaused, error, wordCount
2. Use `useRef<AbortController | null>` for abort management
3. `startStream`: POST to `/api/generate/chapter` with `{ projectId, chapterNumber, adjustments }`, pipe through `TextDecoderStream`, parse SSE lines same as `useOutlineStream`:
   - Skip lines starting with `: ` (OpenRouter heartbeat)
   - Skip non-`data: ` lines
   - Skip `data: [DONE]`
   - Parse JSON: `{ choices: [{ delta: { content: string } }] }` — extract `content`
   - Accumulate content tokens into `accumulated` string
   - Update `streamedText` state with accumulated
   - Update `wordCount` with `accumulated.trim().split(/\s+/).filter(Boolean).length`
4. In catch block: check `err.name === 'AbortError'` and return silently (intentional abort)
5. In finally block: always `setIsStreaming(false)`
6. `pause()`: save current streamedText, abort controller, set isPaused true, set isStreaming false
7. `stop()`: abort controller, set isStreaming false, set isPaused false
8. Before starting a new stream, always abort any existing one first (prevent ReadableStream locked error)
9. Add `'use client'` directive at top of file

Do NOT parse the accumulated text as JSON (unlike outline stream — chapter prose is raw text).
  </action>

  <name>Task 2: Create chapter server actions</name>
  <files>src/actions/chapters.ts</files>
  <action>
Create server actions for chapter prose management. Add `'use server'` directive.

Import `createClient` from `@/lib/supabase/server`, `revalidatePath` from `next/cache`.

Implement three exported functions:

1. `saveChapterProse(projectId: string, chapterNumber: number, text: string)`:
   - Authenticate user, verify project ownership (same pattern as `src/actions/project-memory.ts` verifyProjectOwnership)
   - Upsert into `chapter_checkpoints` table: `{ project_id: projectId, chapter_number: chapterNumber, chapter_text: text }` with onConflict `'project_id,chapter_number'`
   - Do NOT call revalidatePath (this is used for auto-save debounce — revalidation would reset client state)
   - Return `{ success: true }` or `{ error: string }`

2. `getChapterCheckpoints(projectId: string)`:
   - Authenticate user, verify project ownership
   - Select all from `chapter_checkpoints` where `project_id = projectId`, order by `chapter_number asc`
   - Return `{ data: ChapterCheckpointRow[] }` or `{ error: string }`
   - Import `ChapterCheckpointRow` from `@/types/project-memory`

3. `updateProjectWordCount(projectId: string)`:
   - Authenticate user, verify project ownership
   - Query all `chapter_checkpoints` for this project, select `chapter_text`
   - Sum word counts: for each checkpoint, `checkpoint.chapter_text.trim().split(/\s+/).filter(Boolean).length`
   - Count how many checkpoints have non-empty `chapter_text` (this is `chapters_done`)
   - Update `projects` table: `{ word_count: totalWords, chapters_done: chaptersWithText }`
   - Call `revalidatePath(\`/projects/\${projectId}\`)`
   - Return `{ success: true, wordCount: totalWords, chaptersDone: chaptersWithText }` or `{ error: string }`

Use `(supabase as any)` for all table queries (same PostgREST v12 workaround as project-memory.ts).
