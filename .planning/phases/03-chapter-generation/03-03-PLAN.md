Create a `'use client'` component that displays streaming prose with a typing cursor effect, live word count, and pause/stop controls.

**Props:**
```typescript
interface ChapterStreamingViewProps {
  chapterNumber: number
  chapterTitle: string
  streamedText: string
  isStreaming: boolean
  isPaused: boolean
  wordCount: number
  error: string | null
  onPause: () => void
  onStop: () => void
  onResume: () => void
  onRetry: () => void
}
```

**Layout (per CONTEXT.md — side panel alongside outline):**
The streaming view occupies the right panel. Structure:

1. **Header bar:**
   - "Chapter {number}: {title}" — font-semibold
   - Live word count badge (updates during streaming)
   - Streaming status indicator: pulsing green dot when streaming, amber when paused, none when idle

2. **Prose area:**
   - Scrollable container with `prose prose-sm max-w-none` styling
   - Text renders in paragraphs (split streamedText on double newlines `\n\n` to create `<p>` elements)
   - **Typing cursor effect:** Append a blinking cursor character (`|` or use a `<span>` with CSS animation) at the end of the text when `isStreaming` is true. Use `animate-pulse` or a custom `@keyframes blink` animation.
   - Auto-scroll to bottom during streaming: use a ref on a sentinel div at the bottom and call `scrollIntoView({ behavior: 'smooth' })` when streamedText changes (only while isStreaming — don't auto-scroll when user is reading after pause)

3. **Control bar (bottom):**
   - When streaming: "Pause" button (Pause icon) + "Stop" button (Square icon)
   - When paused: "Resume" button (Play icon) + "Stop" button — with a note: "Generation paused. Text is preserved."
   - When error: Error message in red + "Retry" button
   - When idle (stream completed): "Chapter generated. X words." message

Use `lucide-react` icons: `Pause, Play, Square, AlertCircle, Loader2`.

**Key detail on typing effect:** The user explicitly asked for "word-by-word typing effect — tokens appear as if being typed, immersive 'watching a writer' feel". The SSE stream already delivers tokens incrementally which creates this effect naturally. The blinking cursor at the end of text reinforces the illusion. No artificial delay needed — the LLM token rate IS the typing speed.

**Error state:** If `error` is non-null, show an error card with the message and a retry button. Use `toast` from sonner for transient errors if preferred, but the inline error display should be the primary feedback for generation failures.
