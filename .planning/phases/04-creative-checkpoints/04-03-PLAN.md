**1. Create `src/lib/checkpoint/direction-prompt.ts`**

Build the prompt for direction options generation. The prompt must:
- Stay grounded in the approved outline beats (outline immutability decision)
- Produce execution-level options (tone, pacing, emphasis) not plot changes
- Include chapter state diff and next chapter's outline beats as context

```typescript
export const DIRECTION_OPTIONS_SCHEMA = {
  type: 'object' as const,
  properties: {
    options: {
      type: 'array' as const,
      minItems: 2,   // CONTEXT.md locks "2-4 AI-generated options"
      maxItems: 4,   // CONTEXT.md locks "2-4 AI-generated options"
      items: {
        type: 'object' as const,
        properties: {
          id: { type: 'string' as const, description: 'Unique identifier (e.g., "opt-1")' },
          title: { type: 'string' as const, description: 'One-sentence bold hook describing this direction' },
          body: { type: 'string' as const, description: '3-4 sentence paragraph elaborating on tone, pacing, and character focus for this direction' },
        },
        required: ['id', 'title', 'body'] as const,
        additionalProperties: false,
      },
    },
  },
  required: ['options'] as const,
  additionalProperties: false,
}

export function buildDirectionPrompt(
  chapterNumber: number,
  chapterTitle: string,
  chapterSummary: string,
  chapterStateDiff: StateDiff | null,
  nextChapter: OutlineChapter | null,
  genre: string | null,
  tone: string | null
): { systemMessage: string; userMessage: string }
```

System message: You are a creative writing consultant. Generate 2-4 execution-level direction options for the next chapter. Options must stay within the approved outline's plot beats — they control HOW the beats are executed (tone, pacing, character emphasis, scene focus), not WHAT happens. Each option should be distinct and give the author a meaningful creative choice.

User message: Include:
- Just-completed chapter number, title, summary
- State diff highlights (threads advanced/resolved, character changes)
- Next chapter's outline beats, title, summary, characters
- Genre and tone context

If `nextChapter` is null (should not happen — this route is not called for the last chapter), return an error.

**2. Create `src/app/api/generate/direction-options/route.ts`**

Follow the EXACT pattern from `compress-chapter/route.ts`:
1. `export const dynamic = 'force-dynamic'`
2. Auth via `createClient()` + `auth.getUser()`
3. Parse body: `{ projectId, chapterNumber }`
4. Verify ownership
5. Fetch the chapter checkpoint (for state_diff, summary)
6. Fetch the outline (for next chapter's beats)
7. Get API key from user_settings
8. Get model from user_model_preferences where task_type = 'editing' (reuse editing model per research recommendation)
9. Get project identity (genre, tone) from project_memory
10. Build prompt via `buildDirectionPrompt()`
11. Call OpenRouter with `response_format: { type: 'json_schema' }` and the DIRECTION_OPTIONS_SCHEMA
12. Parse response as `{ options: DirectionOption[] }`
13. Save options to chapter_checkpoints via `saveDirectionOptions` server action
14. Return `{ success: true, options: DirectionOption[] }`

Error handling: same pattern as compress-chapter (try/catch around fetch, JSON parse, etc.).

interface DirectionOptionsBody {
