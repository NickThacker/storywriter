  <name>Task 2: End-to-end Creative Checkpoints verification</name>
  <files></files>
  <action>
Human verifies the complete Creative Checkpoints system (Phase 4):
- Post-compression "Chapter ready" toast with Review action button
- Checkpoint panel sliding in alongside chapter text (55%/45% split)
- Chapter stats display (word count, threads, characters, foreshadowing, continuity)
- Approve button (marks chapter as approved, green badge in list)
- Rewrite dialog with guided mode (tone, pacing, character focus) and advanced free-text toggle
- Direction options: 2-4 AI-generated cards after approval
- Custom direction form (structured fields + advanced toggle)
- Direction injection into next chapter generation
- Impact analysis (on-demand) for direction changes
- Affected badges in chapter list
- Novel completion summary for last chapter

**Prerequisites:**
- Apply migration `00004_checkpoint_extensions.sql` in Supabase SQL Editor
- Dev server running at http://localhost:3000
- OpenRouter API key configured in Settings

**Test Flow:**

1. **Navigate to a project with chapters** — go to http://localhost:3000, open a writing-status project
2. **Generate a chapter** (if none exist) — click "Generate Next Chapter"
3. **Verify checkpoint trigger (CKPT-01):**
   - After generation + compression completes, a toast should appear: "Chapter ready — Review checkpoint"
   - Click the "Review" button on the toast — should select the just-generated chapter
   - The chapter list should show "Draft" (amber) badge for the generated chapter

4. **Verify checkpoint panel:**
   - With the generated chapter selected, a checkpoint panel should slide in on the right (45% width)
   - The chapter text should remain visible on the left (55% width)
   - The panel shows chapter stats: word count, threads, characters, foreshadowing, continuity
   - A step indicator (progress dots) should be visible at the top

5. **Verify approve flow (CKPT-02):**
   - Click "Approve & Continue" in the checkpoint panel
   - The chapter list should update: badge changes from "Draft" (amber) to "Approved" (green)
   - The panel should transition to Step 2 (direction options)
   - NOTE: If this is the last chapter, it should show the Novel Completion Summary instead

6. **Verify rewrite flow (CKPT-03):**
   - Go back to a draft chapter (generate another if needed)
   - Click "Request Rewrite" in the checkpoint panel
   - The rewrite dialog should open in GUIDED mode by default
   - Verify three fields: Tone, Pacing, Character Focus
   - Toggle to "Advanced" mode — verify free-text textarea appears
   - Toggle back to "Guided" — verify fields are preserved
   - Enter adjustments and click "Rewrite Chapter" — chapter should regenerate

7. **Verify direction options (CKPT-04):**
   - Approve a non-last chapter
   - Step 2 should show a loading state ("Generating direction options...")
   - After loading, 2-4 direction cards should appear
   - Each card should have a bold title and a descriptive paragraph
   - Click a card — it should highlight with a ring/border
   - Navigate away and back — the same options should appear (cached, no re-fetch)

8. **Verify custom direction (CKPT-05):**
   - In Step 2, click "Or provide your own direction"
   - Verify structured fields: Tone, Pacing, Character Focus
   - Verify "Advanced" toggle reveals free-text textarea
   - Enter custom direction and click "Confirm Direction"
   - Navigate away and back — the saved direction should persist

9. **Verify direction injection:**
   - After saving direction for Chapter N, generate Chapter N+1
   - The generation should incorporate the saved direction (may require reading the generated text to verify tone/focus)

10. **Verify novel completion summary:**
    - If possible, approve the last chapter
    - After approval, should see "Novel Complete" summary with: total chapters, total words, thread stats
    - Should NOT show direction options for the last chapter

**Optional (if time permits):**
11. **Verify impact analysis:**
    - Change a previously saved direction on an earlier chapter
    - Click "Analyze Impact" if available
    - Downstream chapters should show "Affected" badges (amber AlertTriangle)
